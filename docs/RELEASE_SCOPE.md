# نطاق الإصدار 1.2.0 — Pilot تشغيلي نهائي

## داخل النطاق

- ورشة واحدة فعلية.
- تسجيل دخول الموظفين وإدارة حساباتهم وأدوارهم من داخل النظام.
- استقبال سيارة حضورية وإنشاء العميل والسيارة والحجز وأمر العمل في خطوة واحدة.
- إدارة العملاء والسيارات.
- إدارة الخدمات والأسعار والمدد.
- حجز إلكتروني.
- إدارة الحجوزات وأوامر العمل وتفاصيل الاستلام والتكاليف والفني وموعد التسليم.
- حفظ لقطة اسم الخدمة وسعرها ومدتها وقت الحجز.
- تشخيص موثق وعناصر DTC.
- موافقات تشغيلية من الموظف أو بوابة العميل.
- فحص جودة إلزامي قبل الجاهزية.
- مستندات خاصة وبوابة عميل.
- تفعيل حساب العميل من لوحة الإدارة.
- RLS وعزل بيانات الورشة والعميل.
- إعداد Supabase وGitHub وVercel.

## خارج النطاق عمدًا

- SaaS متعدد الورش مكتمل.
- الاشتراكات والباقات والفوترة.
- بوابة دفع.
- WhatsApp Business API وSMS.
- مخزون قطع ومحاسبة كاملة.
- رواتب وحضور.
- تطبيق Android/iOS أصلي.
- وكيل AI تشخيصي داخل النظام.

## معيار النجاح

يُعد الـPilot ناجحًا عندما تعمل دورة فعلية متكررة دون حلول جانبية:

```text
عميل → سيارة → حجز → أمر عمل → تشخيص → موافقة → تنفيذ → جودة → مستندات → تسليم → بوابة العميل
```

تُقاس المشكلات والطلبات من الاستخدام الحقيقي لمدة 30–60 يومًا قبل تحديد المرحلة التالية.
