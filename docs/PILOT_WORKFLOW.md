# سير العمل التشغيلي

## الاستقبال الحضوري

من صفحة **استقبال سيارة** ينشئ موظف الاستقبال العميل والسيارة وحجز الوصول وأمر العمل في عملية واحدة، مع تسجيل العداد والوقود والأولوية وموعد التسليم المتوقع. للـPilot

## 1) العميل والسيارة

يمكن للموظف إنشاء العميل والسيارة من لوحة الإدارة، أو يُنشئهما الحجز العام تلقائيًا. يحتفظ النظام بتاريخ السجلات عند الأرشفة بدل الحذف.

## 2) الخدمات

يدير المدير اسم الخدمة ووصفها وسعرها ومدتها وترتيبها وحالتها. تستخدم صفحة الخدمات ونموذج الحجز نفس البيانات الفعلية من `service_catalog`.

## 3) الحجز

العميل يرسل من `/book`:

- الاسم والجوال.
- الشركة والموديل والسنة واللوحة.
- الخدمة.
- وصف المشكلة.
- الموعد المفضل.

تنفذ `create_public_booking` التحقق وإنشاء/تحديث العميل والسيارة وإنشاء الحجز وسجل الحالة.

مسار الحجز:

```text
new → confirmed → arrived → in_diagnosis → waiting_approval → approved → in_service → completed
```

مع فروع للإلغاء وعدم الحضور.

## 4) أمر العمل

تحويل الحجز إلى أمر عمل عملية ذرية تمنع التكرار وتنسخ العميل والسيارة والشكوى والأولوية.

المسار الأساسي:

```text
open → diagnosing → waiting_approval → approved → in_progress → quality_check → ready → delivered
```

## 5) التشخيص

التقرير مرتبط بالورشة وأمر العمل والعميل والسيارة، ويحتوي:

- شكوى العميل.
- أكواد DTC.
- النتائج والقياسات.
- التوصيات.
- الخلاصة الفنية.
- ملخص العميل.
- حالة التقرير والتحكم في ظهوره.

## 6) الموافقة

ينشئ الموظف طلب موافقة يتضمن:

- المبلغ المطلوب.
- ملخص الأعمال أو العرض.
- قناة التواصل.
- هل يظهر للعميل في البوابة.

يمكن تسجيل القرار بواسطة الموظف أو العميل. تُحفظ القناة والملاحظة ووقت الرد. الموافقة تنقل الأمر من `waiting_approval` إلى `approved`.

## 7) التنفيذ والجودة

بعد الموافقة يبدأ التنفيذ. لا يمكن جعل السيارة جاهزة مباشرة.

```text
in_progress → quality_check
```

في الجودة تُراجع قائمة البنود. عند وجود بند فاشل:

```text
quality_check → in_progress
```

بعد اجتياز جميع البنود:

```text
quality_check → ready
```

ويُسجل وقت الإكمال.

## 8) المستندات

يمكن ربط المستند بالعميل والسيارة والحجز وأمر العمل وتقرير التشخيص. تتحقق API أن جميع الروابط تتبع العميل والورشة نفسيهما.

## 9) بوابة العميل

يفعّل المدير الحساب من صفحة العميل. يرى العميل فقط بياناته وما تم السماح بإظهاره، ويستطيع:

- متابعة الحجوزات وأوامر العمل.
- الموافقة أو الرفض على الطلبات.
- قراءة التقارير المنشورة.
- فتح المستندات المنشورة.
- تغيير كلمة المرور المؤقتة.
